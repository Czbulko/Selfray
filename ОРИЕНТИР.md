# Education — ориентир по проекту

Монорепо на **Tamagui**: один код работает и в вебе (Next.js), и в нативе (React Native / Expo).
Дизайн-система строится **от токенов** (палитра, типографика, отступы, радиусы, тени) — ровно как в Figma Variables.

## Структура

```
education/
├─ apps/
│  ├─ next/     → веб-приложение (Next.js 16). Это то, что деплоится на Vercel для демо.
│  └─ expo/     → нативное приложение (React Native / Expo). Лежит рядом, пока не нужно — не мешает.
├─ packages/
│  ├─ config/   → ДИЗАЙН-ТОКЕНЫ и тема. Сердце дизайн-системы.
│  │            src/tamagui.config.ts — палитра/отступы/радиусы/тени/темы
│  │            src/fonts.ts          — шрифты (типографика)
│  ├─ ui/       → БИБЛИОТЕКА КОМПОНЕНТОВ с вариантами (src/*.tsx)
│  └─ app/      → ЭКРАНЫ и навигация, общие для web и native
│               features/home/screen.tsx        — главный экран
│               features/user/detail-screen.tsx — пример второго экрана
│               provider/                        — провайдеры темы/тостов/safe-area
```

Принцип: **экран пишется один раз** в `packages/app/features/...` и подключается и в `apps/next`, и в `apps/expo`.

## Где что менять

| Хочу… | Файл |
|---|---|
| цвета, отступы, радиусы, тени, тему | `packages/config/src/tamagui.config.ts` |
| шрифты / типографику | `packages/config/src/fonts.ts` |
| добавить компонент в библиотеку | `packages/ui/src/` (+ экспорт в `index.tsx`) |
| новый экран | `packages/app/features/<имя>/screen.tsx` |
| маршрут в вебе | `apps/next/app/...` |
| маршрут в нативе | `apps/expo/app/...` |

## Команды (из корня проекта)

```bash
# веб локально (демо в браузере, http://localhost:3000)
cd apps/next && yarn dev

# собрать веб для прода (то же, что делает Vercel)
yarn web:prod

# нативное приложение (Expo)
yarn native
```

Пакетный менеджер — **yarn 4** (подхватывается автоматически из `.yarnrc.yml`, ставить вручную не надо).

## Деплой на Vercel

В `apps/next/vercel.json` уже лежит конфиг. Деплоить можно двумя путями:
1. Залить репо на GitHub → подключить в Vercel (Root Directory = `apps/next`). Каждый push автодеплоится.
2. Или `vercel` CLI из папки `apps/next`.

## Отдать девам

Отдельного «экспорта» не нужно — это и есть готовая React-кодовая база на их стеке (Tamagui = RN + web).
Девелопер забирает репозиторий как есть: токены в `packages/config`, компоненты в `packages/ui`, экраны в `packages/app`.
